## 14. Tool Selection Guidance

### By Project Characteristic

| Factor | Lightweight | Enterprise |
|--------|-------------|-----------|
| **Project size** | Planner, Lists, Linear | Microsoft Project, Jira, Azure Boards |
| **Methodology** | Planner (Kanban), Linear | Azure DevOps (Agile), MS Project (Waterfall) |
| **Team size** | Planner, Trello, To Do | SharePoint, Project, ServiceNow |

### By Organizational Ecosystem

- **Microsoft 365**: Teams, SharePoint, Planner, Lists, Project, Power BI, Excel, Power Automate — single sign-on, integrated workflows, compliance, data residency
- **Atlassian**: Jira, Confluence — strong for Agile development teams
- **Azure DevOps**: Planning, repos, pipelines, boards, test plans — unified development lifecycle

### When to Use Multiple Tools

- **Planning + execution**: MS Project (strategic) + Planner/Boards (team-level)
- **Product + project**: ProductPlan/Aha! (strategy) + Jira/Boards (delivery)
- **Communication + tracking**: Teams (communication) + Planner/Lists (tracking)
- **Execution + reporting**: Any tracking tool + Power BI (dashboards)
