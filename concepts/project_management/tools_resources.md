## 1. Core PM Resources Overview

### Essential PM Deliverables

Core PM resources needed before selecting tools:

**Strategic & Planning**: Project Charter, Project Roadmap, Project Plan, WBS, Requirements Documentation

**Management & Control**: Risk Register, Stakeholder Register, Issue Log, Change Log, Status Reports

**Resource & Financial**: Resource Plan, Budget & Cost Management, Schedule, Time Tracking

**Communication & Documentation**: Communication Plan, Meeting Minutes, Project Documentation, Lessons Learned

**Quality & Governance**: Quality Plan, Governance Documentation, Compliance Documentation

### Resource Lifecycle

- **Initiation**: Charter, Stakeholder Register, Roadmap
- **Planning**: Project Plan, WBS, Risk Register, Resource Plan, Budget, Schedule
- **Execution**: Status Reports, Issue Log, Change Log, Time Tracking, Meeting Minutes
- **Monitoring & Control**: Status Reports, Budget Tracking, Risk Register updates
- **Closing**: Lessons Learned, Final Reports, Documentation Archive

### Tool Support

- **Planning tools** → Plans, Roadmaps, Schedules, WBS
- **Task management** → Task tracking, progress monitoring
- **Collaboration** → Communication, documentation, knowledge sharing
- **Reporting** → Status reports, dashboards, analytics
- **Resource management** → Resource planning, capacity, time tracking

## 2. ITPM Core Resources Overview

### IT-Specific Resources

Additional resources needed for IT projects beyond core PM resources:

**Technical Documentation**: Architecture diagrams, API specifications, database schemas, system design documents, technical requirements

**Development & DevOps**: Code repositories, CI/CD pipelines, Infrastructure as Code (IaC), deployment scripts, environment configurations

**Testing & Quality**: Test plans, test cases, test results, quality metrics, performance benchmarks, security test results

**Security & Compliance**: Security assessments, threat models, compliance documentation, audit trails, access control documentation, data classification

**Vendor & Integration**: Vendor contracts, API documentation, integration specifications, third-party service agreements, SLA documentation

**Technical Debt**: Technical debt backlog, refactoring plans, architecture improvement roadmap, code quality metrics

**Operations & Support**: Runbooks, operational procedures, monitoring dashboards, incident response plans, disaster recovery documentation

### ITPM Resource Lifecycle

- **Initiation**: Technical feasibility assessment, architecture overview, security requirements
- **Planning**: Technical design, API specifications, infrastructure plan, security assessment, test strategy
- **Execution**: Code repositories, CI/CD pipelines, technical documentation, test results, deployment plans
- **Monitoring & Control**: Performance metrics, security monitoring, technical debt tracking, operational dashboards
- **Closing**: Technical documentation archive, knowledge transfer, production handover, support documentation

### Tool Support for ITPM Resources

- **Development tools** → Code repositories, CI/CD, version control (Azure DevOps, GitHub, GitLab)
- **Architecture tools** → Diagrams, design documentation (Visio, Lucidchart, draw.io)
- **Testing tools** → Test management, automation, quality metrics (Azure DevOps, Jira, TestRail)
- **Security tools** → Security scanning, compliance tracking, vulnerability management
- **Monitoring tools** → Application performance, infrastructure monitoring, dashboards (Azure Monitor, Application Insights)

## 3. Tool Categories & When to Use

### Microsoft 365 Ecosystem

**Integrated suite**: Teams, SharePoint, Planner, Lists, Project for the web, Power BI, Excel, OneDrive  
**When**: Microsoft 365 organizations, enterprise collaboration, integrated workflows  
**Benefits**: Single sign-on, unified collaboration, data integration, compliance, security

### By Project Phase

- **Planning**: Microsoft Project, Project for the web, Lists, Planner, Visio | Smartsheet, Confluence, Miro, Lucidchart
- **Execution**: Teams, Planner, Lists, Project for the web, To Do, SharePoint | Jira, Asana, Monday.com, Slack
- **Monitoring & Control**: Power BI, Excel, Power Automate, SharePoint dashboards | Tableau, project dashboards, time tracking
- **Product Management**: Power BI, Excel, SharePoint, Teams, Planner | ProductPlan, Roadmunk, Aha!, Jira Product Discovery, Pendo

- **Microsoft Project (Desktop)**: Gantt charts, resource allocation, critical path (large projects, Waterfall, enterprise)
- **Project for the web**: Cloud-based PM, team collaboration (mid-size, Agile/hybrid, Microsoft 365)
- **Microsoft Planner**: Kanban boards, task management (simple projects, small to mid-size teams)
- **SharePoint Lists**: Custom tracking, workflow automation (flexible tracking, structured data)
- **Microsoft Visio**: Diagrams, flowcharts, process mapping (process documentation, architecture)
- **Microsoft Whiteboard**: Digital whiteboarding (planning workshops, Teams integration)

**Integration**: All integrate with Teams, SharePoint, Power Automate, Power BI

### Other Planning Tools

- **Smartsheet**: Spreadsheet-based PM, Gantt charts (hybrid, budget tracking)
- **Primavera P6**: Enterprise portfolio management (large programs, construction/engineering)
- **Jira**: Agile planning, sprint planning (development teams, IT projects)
- **Azure DevOps**: Planning, tracking, CI/CD (Microsoft ecosystem, development workflows)

- **Lucidchart**: Diagrams, flowcharts (process documentation, architecture)

## 10. Tool Selection Criteria

### Project Characteristics

- **Project size**: Small → Planner, Lists; Large → Microsoft Project, Project for the web
- **Project type**: Software → Azure DevOps or Jira; Infrastructure → Microsoft Project; Product → Microsoft 365 suite
- **Team size**: Small → Planner, Teams; Large → Microsoft Project, SharePoint, Teams with governance
- **Methodology**: Agile → Azure DevOps, Planner, Project for the web; Waterfall → Microsoft Project; Hybrid → Microsoft 365 suite

### Organizational Factors

- **Microsoft 365 ecosystem**: Teams, SharePoint, Planner, Lists, Project, Power BI, Excel, OneDrive, Power Automate
  - **Benefits**: Single sign-on, unified collaboration, integrated workflows, compliance, security, data residency
  - **When to prioritize**: Microsoft 365 organizations, enterprise collaboration needs, integrated workflows
- **Other ecosystems**: Atlassian (Jira, Confluence); Google (Google Workspace tools)
- **Budget**: Microsoft 365 (bundled) vs. standalone tools; Enterprise vs. lightweight tools
- **Compliance**: Microsoft 365 compliance features (audit trails, security, data residency, compliance center)
- **Integration needs**: Microsoft 365 integrates with Azure, Dynamics 365, Power Platform, CI/CD, ERP, CRM

### Team Factors

- **Team experience**: Familiar tools vs. new tools requiring training
- **Remote/distributed**: Tools with strong collaboration (Teams, Slack, Miro)
- **Technical vs. non-technical**: Technical teams → developer-friendly; Business teams → user-friendly

### When to Use Multiple Tools

- **Planning vs. execution**: Microsoft Project (planning) + Planner/Lists (execution)
- **Product vs. project**: Product tools (ProductPlan, Aha!) for strategy, Microsoft tools for execution
- **Communication vs. tracking**: Teams (communication) + Planner/Lists/Project (tracking)
- **Reporting vs. execution**: Execution tools + Power BI (reporting)

### Tool Consolidation

- **Benefits**: Single source of truth, reduced context switching, integrated workflows
- **When to consolidate**: Similar use cases, overlapping functionality, integration benefits (Microsoft 365 provides integrated suite)
